Aide mémoire des commandes de base Git

À l'exception de la commande servant à cloner un repo, les opérations mentionnées dans ce document nécessitent un compte GitHub.

Cloner un repo

Cloner un repo Git permet de créer une copie exacte d'un repo remote (distant) sur votre machine locale.

git clone REPO_URL DEST_LOCALE

Remplacer REPO_URL par l'adresse de votre repo et DEST_LOCALE par le dossier local à votre machine.

Commit

L'opération Commit permet de renseigner les modifications apportées au code.

git commit -m "Message à renseigner en lien avec la modification"

Vous remarquerez, avec le temps, que certains utilisent des conventions d'écriture.

Par exemple, si le commit est:

  • un ajout de code, on préfixe par ADD
  • une correction de bogue, on préfixe par FIX
  • un travail en cours, on préfixe par WIP

Staging

La commande pour ajouter des fichiers à votre repo avant de commit est: git add FILES

Par exemple, git add . ajoutera tous les fichiers du dossier en cours (noter le point après le add ) qui ne sont pas présents dans le repo.

FETCH

Fetch un repo Git permet de vérifier s'il y a eu des changements depuis la dernière fois que vous avez PUSH comparant votre code avec celui du repo.

git fetch REPO_URL|BRANCH

Remplacer REPO_URL par l'adresse de votre repo et BRANCH par le nom de la branche que vous voulez vérifier. N.B Vous devez préalablement avoir un repo qui est initialisé localement pour utiliser cette commande.

PULL

L'opération Pull permet de récupérer les modifications apportées au projet. Si le commit votre dossier local est < que celui distant (remote ), alors Git mettra à jour votre copie locale (distant vers local).

git pull REPO_URL

N.B si votre dossier local est déjà synchronisé avec votre dossier distant, vous pouvez utiliser uniquement: git pull.

PUSH

L'opération Push vous permet d'envoyer vos modifications vers votre repo (local vers distant).

git push REPO_URL BRANCH

Remplacer REPO_URL par l'adresse de votre repo et BRANCH par le nom de la branche que vous voulez envoyer votre code vers. N.B Vous devez préalablement avoir un repo qui est initialisé localement pour utiliser cette commande.
